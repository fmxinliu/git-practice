(1) git init - 初始化 git 仓库

在执行完git init命令后，会在当前仓库下生成一个 .git 目录，其他目录保持不变。

使用方法

git init <newrepo>

初始化后，会在 newrepo 目录下会生成一个名为 .git 的目录。

(2) git clone - 从现有 git 仓库中拷贝项目（类似 svn checkout）

使用方法

git clone <repo> <directory>

参数说明：

  • repo：原仓库的 url。
  • directory：目标仓库的 url（可省略）。

(3) git status - 查看上次提交之后是否有修改

典型使用：

git status -sb

查看简短修改信息，附带当前所在的分支信息。

(4) git add - 将指定文件由工作区添加到暂存区（类似 svn add）

使用方法：

git status <file1> <file2> ...

file1、file2 可使用通配符。

注意：

与 svn add 不同的是，无论是否新增文件，只要有修改，就必须执行 git add 将文件添加到暂存区，然后才能 git commit 到仓库。

(5) git commit - 将暂存区内容添加到仓库中（类似 svn commit）

典型使用：

git commit -m <message>

message 表示本次提交的修改说明。

(6) git log - 查看提交历史（类似 svn log）

典型使用：

git log --oneline

查看历史记录的简洁的版本。

(7) git reset - 版本回退

典型使用：

git reset --hard <SHA>

回滚仓库到指定的版本（SHA 为git log显示的每次提交的惟一哈希码）。

(8) git reflog - 查看所有版本提交历史

典型使用：

通过git reset回退版本后，该版本之后的提交无法通过git log查到。如果想要撤销回退，可以通过该指令查找对应的哈希，然后执行git reset进行回退。

(9) git checkout - 撤销修改/创建 or 切换分支

典型使用：

git checkout -- <file>撤销工作区 file 文件的修改，回退到暂存区/版本库中的版本。实例1，实例2

git checkout <branch>切换到 branch 分支。

git checkout -b <branch>创建 + 切换到 branch 分支。实例3

(10) git branch - 查看/创建/删除分支

典型使用：

git branch查看所有分支。

git branch <branch>创建 branch 分支（但不会切换到新分支）。

git branch -d <branch>删除 branch 分支。

(11) git tag - 创建/删除标签

典型使用：

git tag查看所有标签。

git tag -a <tagname> -m <commit—info>[创建+]指定标签信息。

git tag -d <tagname>删除指定标签。

git tag -n <tagname>查看指定标签信息。

(12) git remote - 关联/解除关联远程仓库

典型使用：

git remote -v查看关联的远程仓库。

git remote add origin <remote_url>本地仓库关联远程仓库 origin。

git remote rm <remote_name>删除指定的远程仓库的关联。

(13) git push - 向远程仓库推送修改

典型使用：

git push origin <branch_name>推送本地当前分支到远程仓库 origin 指定的分支 branch_name。实例1

git push -u origin <branch_name>将本地的 branch_name 分支推送到远程仓库 origin 的对应分支，同时指定 origin 为默认远程仓库，后面就可以不加任何参数使用 git push 了。实例2

git push把当前分支 master 推送到远程仓库对应的分支。
