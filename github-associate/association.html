github

◆ git 是分布式版本控制系统，每个人的电脑上都是一个完整的版本库。

◆ git 协同开发，理论上只需把各自的修改推送给对方，实际使用通常也有一台充当“中央服务器”的电脑，仅仅用来方便“交换”大家的修改，没有它大家也一样干活，只是交换修改不方便而已。

◆ 充当“中央服务器”的电脑，需要长时间开机，可以直接搭建一台运行 git 的服务器，也可以使用 github 网站提供的免费的 git 远程仓库。

◆ 使用前，请先注册 GitHub 账号，并配置 SSH Key。

关联远程仓库，并推送

◆ 登陆GitHub，创建远程仓库git-practice。

◆ 关联仓库：执行git remote add origin git@github.com:fmxinliu/git-practice.git，关联远程仓库origin。

◆ 推送修改：执行git push -u origin master，把当前master分支推送到远程origin/master分支，以后推送可以使用git push。

◆ 查看关联远程仓库：git remote -v。

◆ 删除关联远程仓库：git remote rm origin。

◆ 查看关联远程分支：git remote show origin。

克隆远程仓库，并拉取分支

◆ 登陆GitHub，查看远程仓库git-practice的地址git@github.com:fmxinliu/git-practice.git git-practice-2。

◆ 克隆仓库：执行git clone git@github.com:fmxinliu/git-practice.git git-practice-2，克隆后，会在本地创建一个仓库git-practice-2。

◆ 进入仓库：执行cd git-practice-2，进入本地仓库。

◆ 查看分支：执行git branch -a，可以查看本地分支 + 远程分支。

◆ 拉取分支：执行git checkout -b build-version origin/build-version，拉取远程分支origin/build-version -> 本地分支build-version，并关联，然后切换到build-version。

      执行git checkout -t origin/commands，拉取远程分支origin/commands -> 本地分支commands，并关联，然后切换到commands。
