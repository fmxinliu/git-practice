github

◆ git 是分布式版本控制系统，每个人的电脑上都是一个完整的版本库。

◆ git 协同开发，理论上只需把各自的修改推送给对方，实际使用通常也有一台充当“中央服务器”的电脑，仅仅用来方便“交换”大家的修改，没有它大家也一样干活，只是交换修改不方便而已。

◆ 充当“中央服务器”的电脑，需要长时间开机，可以直接搭建一台运行 git 的服务器，也可以使用 github 网站提供的免费的 git 远程仓库。

◆ 使用前，请先注册 GitHub 账号，并配置SSH Key。

关联远程仓库并推送

◆ 登陆GitHub，创建远程仓库git-practice。

◆ 仓库关联：执行git remote add origin git@github.com:fmxinliu/git-practice.git，关联后，远程仓库的名字就是origin。

◆ 分支关联：执行git push -u origin master，关联后，以后推送可以使用git push origin master把当前master分支推送到对应的远程master分支。

◆ 查看关联：执行git remote -v

◆ 删除关联：执行git remote rm origin

克隆远程仓库

◆ 登陆GitHub，查看远程仓库 git-practice 的地址。

◆ 克隆仓库：执行git clone git@github.com:fmxinliu/git-practice.git git-practice-2，克隆后，会在本地创建一个仓库git-practice-2。
